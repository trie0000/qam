# QAM — Qualys Asset Management

Qualys VMDR の **AssetGroup / Host / Domain** の登録状況と、その**改廃履歴（追加・変更・削除）**を
ローカルで参照・記録するツール。

- 取り込み：**Qualys API から直接ダウンロード**（プロキシ経由）／一覧 **XML アップロード**の両方
- 差分：前回スナップショットと比較し、**変更だけ**を履歴として蓄積
- 参照：ブラウザ UI（現在の登録状況 / 変更履歴の一覧・フィルタ・Spira 準拠テーブル）
- 記録：**資産単位の作業履歴コメント**を付与（改廃作業時に改廃行から追記）

## アーキテクチャ（TS アプリ + 薄い PowerShell relay）

memola / Spira と同じ構成。**アプリ本体は TypeScript（ブラウザ、esbuild バンドル）**、
**PowerShell は薄い relay**（配信＋プロキシ取得＋ファイル IO）に徹する。配布先 Windows に
追加インストール不要（PowerShell は標準搭載、Node はビルド時のみ）。

```
[qam-start.bat ダブルクリック]
  └→ qam-relay.ps1 が http://127.0.0.1:<port>/ を起動（管理者権限不要・PS 5.1）
       └→ 既定ブラウザで開く → TSバンドル(qam.bundle.js)を配信

  ブラウザ(TS) … XMLパース / 差分計算 / 表示 / フィルタ / テーブル
       │  GET/POST http://127.0.0.1:<port>
       ▼
  qam-relay.ps1（薄い中継・PS 5.1）
       ├ /qam/fetch   : Qualys API を「プロキシ経由 + Basic認証」で取得（CORS/プロキシ回避）
       ├ /qam/file    : UNC 共有のファイル read/write/list（ブラウザが書けない部分）
       └ /qam/bundle  : TSバンドル配信・version
```

ブラウザは UNC への書き込みもプロキシ経由アクセスもできないため relay は必須だが、**重い/変わりやすい
ロジックは全部 TS** に置き、relay は I/O とプロキシ取得だけの最小実装にする。データは UNC 共有
（`QAM_DATA_DIR`）に集約。

### Qualys API ダウンロード（プロキシ経由）
- 接続先 POD・**アカウント/パスワード**・**プロキシ URL** は**設定画面**から登録（既定値は env）。
- relay が `System.Net.Http.HttpClient` + `WebProxy` で Qualys へ（corp GW 参照と同方式）。
  Host 一覧の `WARNING/URL`（`id_min`）ページングは relay 側で follow。
- **認証情報は env かブラウザ設定（localStorage）で保持し、コード/リポジトリには置かない。**

## データ配置（DB なし＝ファイル）

プログラムとデータは分離（データ場所は env）。**指定日の資産状況を表示**できるよう
日次フルスナップショットを保存期間内だけ保持し、**改廃履歴とコメントは永続**で持つ。

```
<QAM_DATA_DIR>/
  snapshots/ group/<date>.json host/<date>.json domain/<date>.json  日次フル状態（保存期間内・現況/指定日参照）
  history/   group.jsonl host.jsonl domain.jsonl                     改廃イベント（追記のみ・永続）
  comments/  comments.jsonl                                          作業履歴コメント（追記のみ・永続）
  runs.jsonl                                                         取込メタ（日付/件数/completed）
  raw/<date>/{group,host,domain}.xml.gz                              生XML（保存期間内）
```

- 同一性キー：AssetGroup=`ID` / Host=`ID`（IP・DNS はキーにしない）/ Domain=ドメイン文字列。
- **保存期間（日）は設定画面から変更可能**。期間を過ぎた `snapshots/*/<date>` と `raw/<date>` を剪定する。
- **改廃履歴 `history/` とコメント `comments/` は剪定しない**（嵩張らず、これが恒久記録）。
- **指定日の資産状況** = その日（無ければ直前）のスナップショットを表示。保存期間より前の日付は
  スナップショットが無いため、改廃履歴ビューで変化を追う。

## 起動・設定

```
1) server/qam.env.example を qam.env にコピーして QAM_DATA_DIR 等を設定
2) qam-start.bat をダブルクリック
3) 開いたブラウザで利用。終了は UI の「終了」アイコン
```

設定は env に一元化（優先順位：引数 > プロセス環境変数 > `qam.env`）。
**API キー等の秘密情報は env のみに置き、リポジトリには絶対に置かない。**

## 実行方法・使い方

> アプリ内のヘルプ（右上の「?」）にも同じ内容があります。

### 起動・終了
- **`qam-start.bat` をダブルクリック** → relay（別ウィンドウ）が起動し、既定ブラウザで自動的に開く。
- 「中継サーバに接続できません」が出たら、relay ウィンドウの起動を確認して「再接続」。
- 終了は UI 右上の「終了」アイコン（relay を停止）。

### 最初の設定（右上の ⚙ 設定）
- **個人設定**：記入者名（操作履歴・メモの作業者）、Qualys アカウント／パスワード、テーマ、文字サイズ。
  ※ Qualys 認証情報・記入者名は各自のブラウザに保存され共有されない。
- **共通設定**：Qualys 接続先 POD、プロキシ URL、保存期間（日）、ライセンス上限、
  ユーザ登録の business_unit／国（country）、自動バックアップ間隔（分）・保管（日）、バックアップからの復元、
  四半期検査の年度開始月・対象の接続点ID パターン、検査登録の既定値（SCAN／MAP のオプションプロファイル・
  スキャナー・タイムゾーン）。
- **開発者**：データのリセット（資産／履歴／メモを種類選択）、登録情報のリセット、ビルド情報。

### データの取込（右上の「取込」）
- **API ダウンロード**：種別（すべて／個別）を選んで Qualys から取得。Host 取込時は IPs in Subscription（VM 限定）も自動取得。
- **XML アップロード**：Qualys の一覧 XML を読み込み（種別は自動判定）。
- **変更履歴 CSV**：手運用の改廃履歴 CSV を取り込み（種別ごと・ヘッダ名で列対応）。
- **値 CSV(AssetGroup)**：接続点 ID をキーに 事業場名／接続名称／拠点名称／コメント／外接番号 を上書き。
- 取込は「取込日時」ごとに保存（同じ日時＝上書き／別時刻＝別取込として追加）。件数が急減した場合は確定前に確認。

### 資産一覧
- 上部タブで種別、「基準（取込日時）」で時点を選択。検索（ID／名前／IP／FQDN、IP はレンジ内判定も）。
- 列名クリックで Excel 風オートフィルタ（並べ替え＋値で絞り込み、Esc で閉じる）。「列表示」で表示／並べ替え／幅変更。
- 事業場名／接続名称／拠点名称／コメント／外接番号 はセルをクリックして手入力（自動保存）。
- **メモ**列はクリックでその場編集、空にすると消去（プレースホルダに戻る）。
- マウスを表の上に置いて **PageUp/PageDown** で 1 ページスクロール。CSV／Excel／全資産 Excel で出力。

### 変更履歴
- 左カレンダーで変更日、期間・種別チップで絞り込み。更新日は JST 表示。
- **行クリック**で「項目｜変更前｜変更後」の 2 カラム表示（追加＝後のみ／削除＝前のみ／変更＝差分を色分け）。
- 接続点 ID・FQDN・IP・所属 AssetGroup 等は**その変更を行った時点の値で固定**（最新情報が変わっても変化しない）。
- **CSV／Excel は各項目の「変更前」「変更後」をそれぞれ列に展開**して出力。
- 追加 IP／削除 IP・追加 FQDN／削除 FQDN 列で増減確認。不要な履歴は選択して削除可。

### 四半期検査

「四半期に一度は SCAN / MAP 検査を実施する」ルールに対して、**現四半期の充足状況**を一覧する。

- **対象**：AssetGroup タイトルから切り出した**接続点ID**（タイトル先頭〜最初の半角スペース。資産一覧の
  「接続点ID」列と同じ切り出し）が対象パターン（既定 `^[A-Z]{2}[0-9]{3,4}D?$` ＝英字2＋数字3〜4＋末尾D任意）に
  一致するもの。**一覧も接続点ID 単位**で出す（同じ ID の AssetGroup は 1 件に束ねる）。
  SCAN は接続点単位、**MAP はその接続点に登録されたドメイン単位**で判定する。
  **ドメイン未登録の接続点は MAP 対象外**。
- **SCAN 対象外の条件**：接続点ID が **D で終わらず、かつ IP（IP_SET）が未登録**の AssetGroup は
  スキャンする実体が無いので SCAN 対象から外す（D 終わりは動的運用で IP を登録しないため対象のまま）。
  この条件で外れても、ドメイン登録があれば **MAP の対象には残る**。
- **判定**：現四半期内に完了した実施があれば「検査済み」、無くても四半期内に次回実行予定がある
  有効なスケジュールがあれば「スケジュール済み」、どちらも無ければ**未対応**。
- **画面**：達成率カード（SCAN / MAP）→ **週次サマリ**（週ごとの実施・予約と累計）→
  **対象×週マトリクス**（接続点ごとに S=SCAN / M=MAP を週列に併記）→ **検査一覧**（実行履歴・予約済み）→
  **対象母集団**（どの接続点が対象／対象外か、その理由）→ **取得内訳（診断）**。
- **取得**：「Qualys から取得」で実施済み/スケジュールの scan・map を取得する（母集団の
  AssetGroup は資産取込のスナップショットを使うので再取得しない）。CSV / Excel で出力可能。
- **取込日スナップショット**：取得結果は `inspection/<日付>.json` に**取込日ごと**に保存する（同じ日に
  取り直したら上書き）。ツールバーの「取込日」で**過去の時点の状況を後から確認**できる。母集団の
  AssetGroup もその日以前の最新スナップショットに合わせるので、当時の登録状況で判定される。
  生 XML を保持しているので、対象パターンや年度開始月を変えると過去分も判定し直される。
  保存期間（共通設定）を過ぎたものは資産スナップショットと同様に剪定される。
- 四半期の区切り（年度開始月）と対象パターンは**共通設定**で変更できる。

### 簡易検査（左ペインの「簡易検査」）

**AssetGroup（とドメイン）を払い出してから、検査スケジュールを登録する**一連の操作をまとめて行う
（**作成のみ**。変更・削除は Qualys の画面で行う）。四半期検査の「新規検査登録」ボタンからも移動できる。

- **資産種別**を最初に選ぶ。
  - **静的（IP 資産）**：検査種別を「MAP と SCAN の両方／MAP 単体／SCAN 単体」から選ぶ。
    検査資産情報は **IP のみ**（行ごとに「単体（IP・CIDR）」と「レンジ（開始〜終了）」を切り替え、複数指定可）。
    AssetGroup の `IP_SET` に登録される。
  - **動的（FQDN 指定）**：検査種別は **SCAN のみ**（MAP 不可）。IP・IP レンジは登録できず、
    検査資産情報は **FQDN のみ**（複数指定可）。AssetGroup の `DNS_LIST` に登録される。
- **命名規則**：AssetGroup 名 = **「外部接続申請番号(仮)」**、ドメイン名 = **「小文字の申請番号 + "." + 地域コード」**
  （例 `EXT-2026-001` → `EXT-2026-001(仮)` / `ext-2026-001.jp`）。`(仮)` は DNS 名に使えないためドメインでは省く。
- **地域区分**（既定）：日本=`jp` / 北米=`na` / 中南米=`la` / 欧州・CIS・中東阿=`eu` / インド=`in` /
  中国・北東アジア=`cn`。共通設定で追加・変更できる（MAP を含むときのみ使用）。
- **スケジュールのタイトル**：既定で **「AssetGroup名_検査予定日(YYYYMMDD)」** が入る（書き換え可。
  申請番号・検査予定日に追従し、手で書き換えた値は上書きしない）。
- **1回のみ実行**：検査は検査予定日に1回だけ実行する。繰り返し（周期）の設定は無く、
  **検査予定日と開始時刻だけ**を指定する。API 上は `occurrence=daily & frequency_days=1 & recurrence=1`
  （1回実行後に自動で無効化）で表現する。タイムゾーンは共通設定の既定値を使う。
- **登録の流れ**：AssetGroup 作成 → （MAP のとき）ドメイン登録 → SCAN/MAP スケジュール登録。
  **同名の AssetGroup / ドメインが既にある場合は「既存を使う / 別名で作る / 中止する」を確認**する。
  途中で失敗しても、そこまでに完了した内容を表示する。
- **登録の入口**：簡易検査ビューの「検査を登録」で**登録モーダル**を開く（ビュー表示時にも自動で開く）。
  入力量が多いため**背景クリックでは閉じない**（キャンセル／×／Esc のみ。Esc は最前面のモーダルだけ閉じる）。
- **入力はセクション構成**：**申請情報**（外部接続申請番号／件名／申請部門／地域区分／申請者=既定は記入者名）→
  **検査対象**（資産種別・検査種別・IP/FQDN 情報）→ **検査スケジュール**（予定日・時間／タイトル）→
  **その他**（備考欄・複数行）→ オプション（トグル）。
  申請部門は AssetGroup の **Division**、件名・申請者・備考は **Comments** に記録される（未入力は送らない）。
- **登録モード**：「Qualys へ登録する」のほかに**「管理表のみ更新」**を選べる。管理表のみ更新は
  Qualys へ一切登録せず、予定を `inspection/manual.jsonl`（追記のみ・剪定しない）に記録する。
  記録した予定は検査一覧に**状態「管理表のみ」**として出て、四半期判定・週次・マトリクスにも
  通常のスケジュールと同格の「予定」として乗る。
- **スキャナー・オプションプロファイルは「オプション設定」**（折りたたみ・既定は非表示）にまとめ、
  開くと共通設定の既定値が入った状態で表示される。
- 送信前に作成内容の一覧を確認するモーダルを挟み、結果は操作履歴と監査ログに残す（管理表のみ更新は
  Qualys API を呼ばないため監査ログ対象外。操作履歴には「検査登録(管理表のみ)」で残る）。
- オプションプロファイル（SCAN/MAP 別）・スキャナー（既定 `External`）・タイムゾーン（既定 `JP`）は
  **共通設定の既定値**が初期表示され、その場で変更できる。
- relay 側は書き込み先を 4 パスに限定している（スケジュール scan/map・AssetGroup・ドメイン）。
- **更新系 API の監査ログ**：**実行者・発行した API（メソッドとURL）・パラメータ・結果**を
  `<QAM_DATA_DIR>/api-audit.log` に必ず出力する（認証情報は書かない）。参照系は対象外。

### 検査一覧（実行履歴・予約済み）

四半期検査ビューに、取得した応答そのままの一覧を表形式で表示する。**「区分」列**で
**実行履歴**（実施済みの scan/map）と**予約済み**（スケジュール）を見分けられ、列名クリックで
片方だけに絞り込める。列は 区分／種別／実施日時・次回予定／状態／接続点ID／対象／タイトル／参照。
**「新規検査登録」ボタン**から登録フォームを開く。

### ライセンス数推移
- 折れ線＝Unique Hosts Scanned（host 一覧から算出）。破線＝ライセンス上限（設定値）。年度ごとに重ね表示。
- **IPs in Subscription**（VM 限定で取得＝VM の Address Management と一致）。
- **IP 重複チェック**（単体×レンジ等の重複検出）、**IP スコープ診断**（VM／CertView／PC 別取得で差分の出所を特定）。

### ユーザ登録（User 一覧の「＋ ユーザ登録」）
- 氏名（全角スペース区切り「姓 名」・英字は半角化）／会社名／メール／検査対象区分（静的・動的）／権限（Scanner・Reader、動的は Reader 固定）／接続点 ID。
- Qualys の `/msp/user.php?action=add` で作成（接続点 ID→所属 AssetGroup、必須項目は共通設定／既定値で補完、SAML はサブスク設定前提で登録メールは送らない）。

### バックアップ・復元（共通設定）
- 起動時に間隔ごと 1 回、**全データを zip で自動退避**（生 XML・ログ・設定は除外）。期限切れは自動削除。
- 「今すぐバックアップ」で手動退避、「バックアップから復元」でその時点の状態へ巻き戻し。

### 操作履歴
- 取込・編集・削除・自動取込などの操作を、作業者と日時（JST）つきで記録。

### 注意
- ファイルサーバ配置で複数人が使う場合、**同時の更新は不可**（取込・編集は 1 人ずつ）。閲覧の同時利用は可。
- 保存期間を過ぎると資産スナップショットは剪定されるが、**変更履歴・メモ・操作履歴・ライセンス推移は恒久保持**。

## 自動定期取込（タスク スケジューラ × Edge ヘッドレス）

無人での定期取込は **`qam-autoingest.ps1`** を Windows タスク スケジューラから起動して行う。
スケジューラを「時刻トリガ」にして、Edge ヘッドレスで自動取込モード
（`http://127.0.0.1:<port>/?autoingest=<種別CSV>`）を開き、1 回の実行で
**（バックアップ →）当日未取込の種別だけを取込 → 完了したら Edge を終了**する。

`qam-autoingest.ps1` の動作：
1. data フォルダを見て、対象種別すべて**当日スナップショットが既にあれば起動キャンセル**（何もしない）
2. relay が未起動なら起動
3. **Edge ヘッドレス**で `?autoingest=<種別>` を開く（専用 `--user-data-dir` プロファイル）
4. 当日スナップショットが揃う／上限時間（既定 40 分）まで監視して Edge を終了

完了の記録は `<QAM_DATA_DIR>/autoingest.log` に残る。

### 事前準備①：認証情報を専用 Edge プロファイルに保存（1 回だけ）

ヘッドレス Edge は専用プロファイルの localStorage の認証情報を使う（**パスワードを env/リポジトリに置かない**）。
最初に 1 度だけ手動で保存する。

1. `qam-start.bat` を起動（relay を立てる）。
2. 専用プロファイルで Edge を開く（`<port>` は qam.env の `QAM_RELAY_PORT`、既定 18090）。

   PowerShell の場合（先頭に呼び出し演算子 `&`、env は `$env:`）:
   ```powershell
   & "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" "--user-data-dir=$env:LOCALAPPDATA\qam-edge-profile" "http://127.0.0.1:<port>/"
   ```
   cmd の場合:
   ```bat
   "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --user-data-dir="%LOCALAPPDATA%\qam-edge-profile" http://127.0.0.1:<port>/
   ```
3. ツールの設定で Qualys アカウント/パスワードを入力して保存 → Edge を閉じる。

### 事前準備②：手動でテスト実行

```
powershell -NoProfile -ExecutionPolicy Bypass -File "<配置先>\qam-autoingest.ps1"
```

`<QAM_DATA_DIR>\autoingest.log` に「Edge 起動 → 取込 → 完了」が出れば成功。

### タスク登録手順

1. **タスク スケジューラ**を開く →「**タスクの作成**」
2. **全般**：名前 `QAM 自動取込` ／ **「ユーザーがログオンしているときのみ実行する」**
   （UNC 共有と LOCALAPPDATA の Edge プロファイルに届かせるため。ログオフ実行不可）
3. **トリガー**：毎日、取込したい時刻
4. **操作**：「プログラムの開始」
   - プログラム/スクリプト：`powershell.exe`
   - 引数：`-NoProfile -ExecutionPolicy Bypass -File "<配置先>\qam-autoingest.ps1"`
5. **設定**（任意）：「スケジュールされた時刻に開始できなかった場合、すぐに開始」をオン

取込種別やプロファイル場所は引数で変更可：
`-File "...\qam-autoingest.ps1" -Kinds "host,group,domain" -ProfileDir "D:\qam-edge"`

> 注意：PC の電源が入っている時刻に設定する（スリープ運用ならトリガ条件で「スリープ解除」を有効化）。
> user 取込は数分〜（最大 30 分前後）かかるため、上限時間 `-MaxWaitSec` は長めに保つ。
